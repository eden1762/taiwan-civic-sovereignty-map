# Taiwan Civic & Sovereignty Map 維護檢查｜2026-06-25 23:30

本次檢查聚焦在首頁地圖穩定性、公開資料查證、手機速度、雙語完整性、活動狀態與維護規則。

## 已確認現況

- 首頁已使用 OpenStreetMap 相關地圖呈現，並有地圖載入失敗時的 fallback 卡片。
- 資料來源清單已集中於 `data.js`，目前涵蓋政府登記、政黨官網、基金會、智庫、活動平台、新聞、事實查核與地圖來源。
- 目前主要資料與文案採單一首頁、單一資料檔、單一互動檔、單一樣式檔架構，未發現需要另建部署子資料夾的必要。

## 本輪建議優先修正

### 1. 進一步降低外部地圖元件風險

目前頁面仍會載入外部地圖樣式與外部地圖元件。如果任一外部資源連線不穩，雖然已有 fallback，但使用者仍可能先看到短暫空白。建議下一輪將「資料清單」視為第一層畫面，地圖視為加分層：

- 頁面一開就先渲染搜尋、分類、資料卡。
- 地圖延後載入，載入成功才顯示互動地圖。
- 地圖載入超過 2.5 秒就切換為「資料清單優先」模式。
- 地圖失敗時，保留搜尋、分類、資料卡、查證來源，不讓畫面卡住。

### 2. 加強位置精準度欄位

每筆組織或活動建議固定補齊：

- `locationQuality`：公開地址已定位、公開範圍已定位、待覆核、僅有城市層級。
- `areaType`：點位、路線、行政區、活動範圍、入口頁。
- `sourceCheckedAt`：最後查證日期。
- `officialSourceLevel`：政府登記、官網、公開活動頁、新聞、社群線索。

### 3. 活動資料狀態規則

維持以下規則：

- `ended` 或結束日期早於今日：不顯示。
- `planning`、`upcoming`、`active`、`portal`：可顯示。
- 只有入口頁不可寫成單一活動，要標示為「活動入口」。
- 位置不確定時不可硬放精準點，需標示待覆核。

### 4. 中文文案方向

建議中文使用：

- 「公開資料」、「查證來源」、「守法參與」、「活動入口」、「公開位置」、「活動範圍」、「待覆核」。
- 可少量使用「資訊不迷路」、「先查再出發」、「手機看也順」等年輕化語氣。
- 避免在使用者可見文案出現開發流程、部署、程式或外部平台技術名詞。

### 5. 英文文案方向

建議英文使用：

- public data, verification sources, lawful civic participation, public location, activity area, source check, privacy-aware.
- 可少量使用 easy to verify, no frozen screen, mobile-first, civic map for real-world action.
- 避免使用工程維護、部署、程式開發者語氣。

## 下輪可直接執行的修改清單

1. 將地圖區預設文案改成「資料先顯示，地圖載入後自動補上」。
2. 增加 `sourceCheckedAt` 與 `officialSourceLevel` 欄位範例。
3. 讓地圖圖例在 fallback 模式下改成「資料清單模式」。
4. 增加資料品質 badge：政府登記、官網、活動頁、新聞、社群線索。
5. 增加手機版快速切換：全部、政黨、倡議、基金會、活動、新聞查核。

## 風險提醒

- Facebook、Instagram、Threads、LINE、Google Maps 等平台常有登入、反爬或顯示限制；只能作為線索或外部連結，正式資料仍需回到政府登記、官網、公開活動頁或新聞來源查證。
- 大量 marker 會拖慢手機，建議分批顯示與分類切換，不一次渲染所有資料。
- 活動範圍若涉及遊行或集會，應以公開公告、交通管制或主辦方公開頁為準，不推測私人集合點。
